# Работа с системой непрерывной интеграции Jenkins. Часть 2

## I. Сборка запускаемого `.exe`-файла для Windows и его распространение через интерфейс Jenkins

1. Скопируйте в свою учётную запись на GitHub [репозиторий](https://github.com/rofrol/swing-demo) или сделайте его fork в своей учётной записи.
2. Разместите приложенный к данной странице gradle wrapper в репозитории и исправьте в файле `.gitignore` список исключений таким образом, чтобы необходимые для выполнения сборки файлы gradle wrapper не игнорировались системой управления версиями. Зафиксируйте изменения и отправьте их в удалённый репозиторий.
3. Создайте проект в [Jenkins](http://jenkins.ivt.yarsu.ru/) для сборки кода из данного репозитория. Используйте имя Вашей учётной записи в качестве префикса имени проекта в Jenkins. Запустите сборку проекта в Jenkins и добейтесь, чтобы проект собирался без ошибок.
4. С помощью плагина [gradle-launch4j](https://github.com/TheBoegl/gradle-launch4j) настройте сборку `.exe`-файла из кода репозитория. Во избежание возможных проблем не рекомендуется добавлять иные опции, кроме `mainClassName`. Проверьте правильность сборки на локальной машине. Зафиксируйте изменения и отправьте их в удалённый репозиторий.
5. Настройте проект в Jenkins таким образом, чтобы ссылка на созданный после сборки `.exe`-файл выводилась на странице проекта. **Указание.** Используйте действие Archive the artifacts в разделе действий, выполняемых после сборки. Путь к публикуемому файлу указывается относительно корня рабочего пространства, посмотреть содержимое которого можно по ссылке Workspace на странице проекта.
6. Скачайте созданный артефакт и проверьте его работоспособность (по возможности).
